# 영상 기반 웹페이지 구현 명세서

## 1. 구현 목표

영상에 등장하는 감성적인 데이트 신청 웹페이지를 동일한 분위기와 흐름으로 재현한다.
전체 콘셉트는 **핑크 파스텔 배경 + 중앙 카드 UI + 귀여운 데이트 신청 플로우**이다.

사용자는 아래 순서로 웹페이지를 경험한다.

1. 데이트 신청 화면
2. `YES` 클릭 시 성공 메시지
3. 날짜/시간 선택 화면
4. 음식 선택 화면
5. 최종 확정 메시지 화면

`no` 버튼은 사용자가 누르기 어렵게 마우스 접근 시 위치가 바뀌는 장난스러운 인터랙션을 가진다.

---

## 2. 전체 화면 스타일

### 화면 비율

* 모바일 웹이 아니라 데스크톱 브라우저 기준
* 화면 중앙에 카드가 배치됨
* 영상에서는 MacBook 화면으로 보이며, 웹페이지 자체는 넓은 데스크톱 캔버스에서 동작

### 배경

* 전체 배경은 연한 핑크 계열 그라데이션
* 색상 예시:

  * 상단/중앙: `#f7dce8`
  * 외곽: `#ead8ef`
  * 밝은 부분: `#fff4f7`
* 부드러운 radial-gradient 느낌
* 화면에 아주 작은 하트 또는 꽃 모양 장식이 몇 개 떠 있는 듯한 느낌을 추가해도 좋음

예시 CSS:

```css
body {
  margin: 0;
  min-height: 100vh;
  background:
    radial-gradient(circle at center, #fff6f8 0%, #f7dce8 45%, #ead8ef 100%);
  display: flex;
  align-items: center;
  justify-content: center;
  font-family: Georgia, "Times New Roman", serif;
  color: #5b2636;
}
```

## 3. 공통 카드 UI

모든 단계는 중앙에 흰색 또는 아주 연한 핑크색 카드로 표시한다.

### 카드 스타일

* 가로 약 `620px`
* 세로는 콘텐츠에 따라 `260px ~ 420px`
* 배경: `rgba(255, 255, 255, 0.72)` 또는 `#fff7fa`
* border-radius: `24px`
* box-shadow:

  * 부드러운 그림자
  * 예: `0 18px 45px rgba(120, 60, 90, 0.15)`
* padding: `44px 48px`
* 중앙 정렬
* 전체적으로 몽글몽글하고 부드러운 인상

```css
.card {
  width: 620px;
  max-width: calc(100vw - 40px);
  padding: 44px 48px;
  border-radius: 24px;
  background: rgba(255, 255, 255, 0.72);
  box-shadow: 0 18px 45px rgba(120, 60, 90, 0.16);
  text-align: center;
}
```

## 4. 폰트/텍스트 스타일

### 메인 제목

* serif 계열
* 굵고 감성적인 느낌
* 색상: 진한 와인/브라운 계열
* 예시 색상: `#5a2234`
* 크기:

  * 메인 질문: `32px ~ 38px`
  * 최종 메시지: `30px ~ 36px`
* font-weight: `700`

```css
.title {
  font-family: Georgia, "Times New Roman", serif;
  font-size: 36px;
  line-height: 1.25;
  color: #5a2234;
  font-weight: 700;
}
```

### 보조 텍스트

* 크기: `15px ~ 18px`
* 색상: `#6d3a4a`
* italic 사용 가능

```css
.subtitle {
  font-size: 16px;
  color: #6d3a4a;
}
```

## 5. 화면 1: 데이트 신청 화면

### URL/라우트

* `/`
* 또는 `/date-start`

### 구성

중앙 카드 안에 다음 요소가 순서대로 들어간다.

1. 상단 프로필 이미지
2. 메인 질문 텍스트
3. 버튼 2개: `YES ♥`, `no`

### 프로필 이미지

* 영상에서는 강아지 얼굴 이미지가 중앙 상단에 있음
* 정사각형 이미지
* 사이즈: `110px x 110px`
* border-radius: `20px`
* 배경은 연보라/회색 톤
* 실제 이미지가 없으면 pug/dog placeholder 사용

```css
.avatar {
  width: 110px;
  height: 110px;
  object-fit: cover;
  border-radius: 20px;
  margin-bottom: 24px;
}
```

### 메인 문구

텍스트:

```text
🌸 Will you go on a date with me? 🌸
```

* 두 줄로 자연스럽게 줄바꿈되어도 됨
* 중앙 정렬
* 글씨는 크고 굵게

### 버튼

#### YES 버튼

* 핑크색 pill 버튼
* 텍스트: `YES ♥`
* 배경: `#d85d93` 또는 `#d86b9c`
* 글자색: white
* border-radius: `999px`
* width: `130px`
* height: `52px`
* hover 시 살짝 커지거나 밝아짐

#### no 버튼

* 보라색 pill 버튼
* 텍스트: `no 🐾`
* 배경: `#9b58c9`
* 글자색: white
* border-radius: `999px`
* width: `110px`
* height: `52px`

### no 버튼 인터랙션

영상에서 `no` 버튼은 마우스가 접근하면 위치가 바뀌는 장난스러운 버튼처럼 보인다.

구현 요구사항:

* `no` 버튼에 hover 또는 mouseenter 발생 시 버튼 위치를 랜덤하게 이동
* 카드 내부 또는 화면 내부에서만 이동
* 사용자가 쉽게 클릭하지 못하도록 함
* position은 `absolute`로 처리 가능
* 이동 시 transition을 `0.2s` 정도 적용

예시 동작:

```js
function moveNoButton() {
  const x = Math.random() * 300 - 150;
  const y = Math.random() * 180 - 90;
  noButton.style.transform = `translate(${x}px, ${y}px)`;
}
```

### YES 클릭 시

* 다음 화면으로 이동
* 상태값: `step = "accepted"`

## 6. 화면 2: YES 클릭 후 성공 메시지

### 구성

중앙 카드에 큰 아이콘과 메시지를 표시한다.

### 아이콘

영상에서는 체크 표시 느낌의 아이콘이 보임.

예시:

```text
✅
```

또는 귀여운 봉투/하트 아이콘 사용 가능.

### 텍스트

메인 텍스트:

```text
WAIT YOU ACTUALLY SAID YES?? 🤭
```

보조 텍스트:

```text
I knew you liked me back :)
```

### 버튼

버튼 텍스트:

```text
pick a date 💗
```

* 핑크색 pill 버튼
* 클릭 시 날짜/시간 선택 화면으로 이동

## 7. 화면 3: 날짜/시간 선택 화면

### URL/라우트

* `/date`

### 구성

중앙 카드 안에 다음 요소가 들어간다.

1. 상단 아이콘
2. 제목
3. 날짜 input
4. 시간 select
5. 제출 버튼

### 상단 아이콘

영상에서는 달력 + 발자국 느낌의 아이콘이 있음.

예시:

```text
📅🐾
```

### 제목

```text
So... when are you free?
```

### 날짜 필드

라벨:

```text
Pick a Day 📅
```

input:

* type: `date`
* placeholder는 브라우저 기본값
* 영상에서는 `mm/dd/yyyy` 형태로 표시됨
* width: 100%
* height: `48px`
* border: `1px solid #c8a8b5`
* border-radius: `8px`

### 시간 필드

라벨:

```text
What Time? ⏰
```

select placeholder:

```text
Select a time...
```

선택 옵션 예시:

```text
1:00 PM
1:30 PM
2:00 PM
2:30 PM
3:00 PM
3:30 PM
4:00 PM
4:30 PM
5:00 PM
5:30 PM
6:00 PM
6:30 PM
7:00 PM
7:30 PM
8:00 PM
```

영상에서는 드롭다운을 열었을 때 여러 시간 옵션이 아래로 표시된다.

### 제출 버튼

텍스트:

```text
set the date! ♥
```

* 핑크색 긴 버튼
* width: 100%
* height: `52px`
* border-radius: `999px`
* 날짜와 시간이 모두 선택되면 다음 화면으로 이동

## 8. 화면 4: 음식 선택 화면

### URL/라우트

* `/food`

### 구성

중앙 카드에 음식 선택 그리드를 표시한다.

### 제목

```text
What are we feeling? 🍽️✨
```

### 보조 텍스트

```text
pick your vibe
```

### 음식 카드 그리드

* 2행 x 3열
* 각 카드 크기: 약 `140px x 120px`
* 카드 배경: 아주 연한 회색/핑크
* border-radius: `16px`
* box-shadow 또는 subtle border
* hover 시 살짝 올라가거나 테두리 강조
* 선택된 카드에는 핑크색 테두리 표시

음식 옵션:

1. Pizza

   * 이모지: `🍕`
2. Sushi

   * 이모지: `🍣`
3. Burgers

   * 이모지: `🍔`
4. Pasta

   * 이모지: `🍝`
5. Tacos

   * 이모지: `🌮`
6. Ramen

   * 이모지: `🍜`

영상에서는 `Ramen` 카드가 선택된 상태로 보이며, 핑크색 테두리가 강조되어 있다.

### 클릭 동작

* 음식 카드 클릭 시 선택 상태 표시
* 선택 즉시 최종 화면으로 넘어가거나, 짧은 딜레이 후 넘어가도 됨
* 영상 흐름상 음식 선택 후 최종 메시지 화면으로 이동

## 9. 화면 5: 최종 메시지 화면

### 구성

중앙 카드에 최종 확정 메시지를 표시한다.

### 메인 텍스트

```text
glad you didn't say no. be ready
by 6, I'm coming to get you 🚗
```

* 영상에서는 두 줄로 표시됨
* 매우 큰 serif font
* 진한 와인색
* 중앙 정렬

### 보조 텍스트

```text
P.S. normal people text. I made a website on Replit, during lunch, for you.
no big deal.
```

* italic 스타일
* 작은 글씨
* 중앙 정렬
* 메인 텍스트 아래 배치

### 하트 장식

```text
♥ ♥ ♥ ♥ ♥
```

* 핑크색
* 보조 텍스트 아래에 배치

### Replit 배지

영상 하단에는 `Built with Replit` 주황색 배지가 있음.

구현 옵션:

* 실제 링크가 없으면 버튼처럼만 구현
* 텍스트:

```text
Built with Replit
```

* 배경: `#f26b2b`
* 글자색: white
* border-radius: `999px`
* 좌측에 Replit 로고처럼 작은 원형 아이콘 또는 단순 `●` 사용 가능

## 10. 전체 상태 플로우

프론트엔드는 단일 페이지 앱으로 구현해도 된다.

상태값 예시:

```js
const [step, setStep] = useState("ask");
const [selectedDate, setSelectedDate] = useState("");
const [selectedTime, setSelectedTime] = useState("");
const [selectedFood, setSelectedFood] = useState("");
```

상태 흐름:

```text
ask
  └─ YES 클릭
      ↓
accepted
  └─ pick a date 클릭
      ↓
date
  └─ 날짜 + 시간 선택 후 set the date 클릭
      ↓
food
  └─ 음식 선택
      ↓
final
```

## 11. 애니메이션 요구사항

### 카드 등장 애니메이션

각 화면 전환 시 카드가 부드럽게 등장해야 한다.

```css
.card {
  animation: popIn 0.35s ease;
}

@keyframes popIn {
  from {
    opacity: 0;
    transform: translateY(12px) scale(0.98);
  }
  to {
    opacity: 1;
    transform: translateY(0) scale(1);
  }
}
```

### 버튼 hover

* 살짝 커짐
* 그림자 증가

```css
button:hover {
  transform: translateY(-2px) scale(1.03);
}
```

### 음식 카드 hover

```css
.food-card:hover {
  transform: translateY(-4px);
  border-color: #d86b9c;
}
```

## 12. 반응형 요구사항

### 데스크톱

* 카드 width: `620px`
* 중앙 정렬

### 모바일

* 카드 width: `calc(100vw - 32px)`
* padding: `28px 22px`
* 음식 그리드는 2열로 변경 가능
* 제목 폰트는 `28px` 정도로 축소

```css
@media (max-width: 640px) {
  .card {
    width: calc(100vw - 32px);
    padding: 28px 22px;
  }

  .title {
    font-size: 28px;
  }

  .food-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}
```

## 15. 최종 구현 시 가장 중요한 포인트

1. 전체 분위기는 “귀엽고 부끄럽고 장난스러운 데이트 신청 웹앱”이어야 한다.
2. 카드 UI는 중앙에 크게 배치한다.
3. 배경은 연한 핑크/보라 그라데이션으로 부드럽게 만든다.
4. 메인 폰트는 serif 계열로 감성적인 느낌을 살린다.
5. `no` 버튼은 마우스를 피하는 인터랙션이 반드시 있어야 한다.
6. 음식 선택 화면에서는 선택된 카드에 핑크 테두리를 표시한다.
7. 최종 화면 문구는 영상과 동일하게 크게 표시한다.
